# kit-mcp

MCP-сервер для работы с документацией UI-компонентов.

## Установка

Используй инсталлер:

```bash
npx github:puryganikita/dev-factory
# Выбери: Extension → kit-mcp
```

Инсталлер автоматически:
- Скопирует MCP-скрипт в `ai-dev-factory/kit-docs-mcp.ts`
- Добавит сервер `kit-docs` в `.cursor/mcp.json`
- Скопирует скилл `kit-components-usage` в `.cursor/skills/`
- Попросит указать путь до документации и сохранит в `ai-dev-factory/dev-factory.config.json`

## Ручная настройка

### Конфиг

В `ai-dev-factory/dev-factory.config.json` укажи путь до папки с документацией:

```json
{
  "extensions": {
    "kit-mcp": {
      "docsDir": "./path/to/kit-docs"
    }
  }
}
```

`docsDir` — относительный путь от корня проекта.

## Формат документации

MCP ожидает папку с `.md`-файлами следующей структуры:

```
docs/
├── overview.md        — реестр всех компонентов
├── Button.md          — документация компонента Button
├── TextField.md       — документация компонента TextField
└── ...
```

### overview.md

Реестр компонентов. Каждый компонент — отдельная секция с заголовком `## Name`:

```markdown
## Button

Кнопка для выполнения действий. Поддерживает варианты primary, secondary, ghost.

## TextField

Текстовое поле ввода. Поддерживает label, placeholder, валидацию.

## Accordion

Раскрывающийся блок для скрытия/показа контента.
```

MCP использует заголовки `## Name` для парсинга списка компонентов и поиска.

### {ComponentName}.md

Полная документация по компоненту. Имя файла должно **точно совпадать** с именем в заголовке `## Name` из `overview.md`.

Рекомендуемое содержание:
- Props (имя, тип, описание, значение по умолчанию)
- Область применения
- Примеры использования
- Ограничения / что запрещено

## Инструменты MCP

| Инструмент | Описание |
|---|---|
| `list_kit_components` | Возвращает `overview.md` целиком — реестр всех компонентов |
| `get_kit_component_doc` | Возвращает `{Name}.md` — полную документацию по компоненту |
| `search_kit_components` | Поиск по ключевому слову в `overview.md` |
